# web-portal-hub

## 项目简介

web-portal-hub 是一个用于归档和发布多个独立 HTML 页面的仓库。本项目不针对任何单个域名或具体网站，而是作为一个通用容器，集中存放各种静态页面资源。每个页面均可独立访问，适合用于演示、存档或轻量级发布场景。

## 目录结构

```
web-portal-hub/
├── pages/           # 存放所有 HTML 页面文件
│   ├── page-1/      # 示例页面目录（可根据需要命名）
│   ├── page-2/
│   └── ...
├── assets/          # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── index.html       # 仓库首页（可选，可列出页面索引）
└── README.md        # 本文件
```

## 页面归档说明

- 所有页面文件统一放置在 `pages/` 目录下，每个页面可单独占用一个子目录，以便管理关联资源。
- 页面之间相互独立，无依赖关系，可直接通过浏览器打开对应 HTML 文件访问。
- 如需引用公共资源，请将文件放入 `assets/` 目录，并在页面中通过相对路径引用。
- 仓库不预设任何特定主题或用途，页面内容由贡献者自行决定。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 将新的 HTML 页面及其资源放入 `pages/` 下的新目录。
3. 如有需要，可更新 `index.html` 以添加页面索引链接。
4. 通过 GitHub Pages 或其他静态托管服务部署本仓库，即可在线访问所有页面。

## 维护说明

- 欢迎提交 Pull Request 添加新的页面或改进现有页面。
- 请确保新增页面不包含恶意代码、非法内容或侵犯他人权益的信息。
- 建议为每个页面添加简要的说明注释（如页面用途、作者、更新日期等）。
- 公共资源请尽量保持通用性和可复用性，避免与特定页面强耦合。
